# Promotion kit — Vibe Playlists

Drafts to announce [Vibe Playlists](https://github.com/craiglush/navidrome-vibe). All copy is
humble, technical, self-hoster-to-self-hoster, and **author-disclosed** — tweak to taste.

## Assets
| File | Where | Notes |
|------|-------|-------|
| `devto-launch.md` | dev.to | The anchor article — write this first, link the rest to it |
| `reddit-navidrome.md` | r/navidrome + Navidrome forum/Discourse | Navidrome-specific |
| `reddit-selfhosted.md` | r/selfhosted | Strict self-promo rules — read them first |
| `discord-lemmy.md` | Navidrome Discord, Lemmy c/selfhosted | Short blurbs |

## The GIF is the most important asset
Most of these posts live or die on a 5–10s GIF of *typing a vibe → a playlist appearing*.
Record it first (storyboard in `repo-polish.md`), add it to the repo README and the dev.to post.

## Suggested order & timing
1. **Polish the repo** (About, topics incl. `navidrome-plugin`, social preview, README GIF) — do this *before* any post so visitors land on something finished.
2. **dev.to article** — the canonical write-up; everything else links to it.
3. **r/navidrome** + **Navidrome Discord** — your home crowd; lowest risk, best early feedback.
4. **Lemmy c/selfhosted**, then **r/selfhosted** — wait for r/selfhosted's allowed self-promo window/flair (see that file); space these out a day or two.
5. **Show HN** — optional, once the GIF + README are solid and you can be around to answer comments.

## Anti-self-promo rules of thumb (so you don't get removed)
- Lead with the story/value, not the link. Disclose you're the author ("I built…").
- One link, near the end. No marketing voice. Answer every comment.
- Don't cross-post the identical text everywhere on the same day.
- Be honest it's early/solo and what *doesn't* work yet (analyzer must run first; plugin needs Navidrome 0.60+).

## Verify before posting
- Re-check AudioMuse-AI's current description/stars before citing it (positioning is *fair*, not dismissive).
- Confirm the exact Navidrome version wording (plugins need 0.60+; you tested on 0.62).
- Make sure the repo README's quick-start actually works for a fresh cloner.
